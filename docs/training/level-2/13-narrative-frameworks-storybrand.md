# Chapter 13 — Narrative Frameworks for Technical Documentation

> **Track:** LEVEL_2 · **Reading level:** 11th–12th grade plain English · **Release:** `v0.1.0-ga-epic17`  
> **Primary route:** `/docs/training/level2-practitioner-index` · **Lab IDs:** DOC-001

> **Note:** board-writer may clarify practitioner docs. It must not draft ads, claim market leadership, or invent APIs.

## Why this chapter matters

Technical docs teach faster when they follow a clear story: problem, guide, plan, proof. This is structure for runbooks — not marketing copy.

## Learning objectives

When you finish, you can:

- Map StoryBrand elements to technical sections.
- Write a short problem → procedure → verification stub.
- Stay inside board-writer boundaries (no campaign copy).

## How to get there

1. Open any Level 2 chapter you finished.
2. Label its problem, plan, and verification sections.
3. Rewrite one dense sentence into two shorter ones.
4. Add a verification checkbox tied to a real route or test file.

## Lab — Runbook structure (DOC-001)

1. Write a five-line stub with Problem, Prerequisites, Procedure, Verification, Sources.
2. Use a real route from this track in the procedure.
3. Add one fail-closed failure note (example: bridge 502).
4. Confirm you did not write website hero or LinkedIn pitch language.
