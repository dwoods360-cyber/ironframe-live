# Dashboard & Core features (student)

**Track:** Student · **Reading level:** 11th grade · **Primary routes:** `/integrity`, `/evidence`, `/cockpit`

Daily Core work: read financial protection posture, keep the hazard pipeline clear, and use sealed evidence for audits.

---

## 1. Integrity Hub and ALE

**Integrity Hub** (`/integrity`) turns security issues into business numbers using **ALE** — estimated dollar loss in one year if a risk is exploited.

Money is stored as **whole cents** so totals stay exact.

### Labels you may see

| Element | Exact label |
|---------|-------------|
| Agent monitor | **AGENT STATUS PULSE** |
| Nav chip | **INTEGRITY HUB** |
| Emergency control | **FREEZE COMMAND POST** |
| Insurance export | **Export Tabular Ledger Data (CSV)** |

**Class goal:** Understand what the hazard pipeline shows. When a hazard appears, notice how posture updates.

Classroom seed baselines (training fixtures only):

| Seed tenant | Example protection target |
|-------------|---------------------------|
| Medshield | $11,100,000.00 |
| Vaultbank | $5,900,000.00 |
| Gridcore | $4,700,000.00 |

## 3. Evidence Vault

1. Open `/evidence`.
2. Find a sealed (or sealable) record your trainer points to.
3. Remember **WORM**: after seal, the record is not deleted or quietly edited.

---

## 4. Ironbloom (energy) panel

If your classroom build shows **Ironbloom** / sustainability:

1. Open the sustainability panel from Integrity Hub.
2. Enter consumption with a **unit** (for example `1500 kWh` or `450 L`).
3. Confirm the UI accepts the entry or shows a clear validation error.

Bare dollar amounts without units should be rejected.
