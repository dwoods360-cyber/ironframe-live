# StoryBrand Framework — Ironframe Application

Donald Miller's **Building a StoryBrand** and **Marketing Made Simple** are primary references for **board-marketing-mgr** and narrative structure for **board-writer**. This document operationalizes SB7 for Ironframe GRC without inventing customer proof.

## Core principle

> If you confuse, you lose. The customer is the hero; Ironframe is the guide.

## SB7 — Seven-part story framework

| # | Element | Ironframe guide role |
|---|---------|---------------------|
| 1 | **Character** (hero) | CISO, GRC director, CFO, or MSSP operator who owns board-grade risk narrative |
| 2 | **Problem** | External: audit/regulator pressure. Internal: fear of board embarrassment. Philosophical: compliance theater is not real risk management |
| 3 | **Guide** | Ironframe — empathetic ("we've seen heatmap GRC fail audits") + authoritative (TAS, BigInt ALE, Irongate, release evidence) |
| 4 | **Plan** | 3-step plan: (1) Connect telemetry via Irongate, (2) Quantify exposure in Command Center, (3) Export audit-ready board materials |
| 5 | **Call to action** | Direct: book a Command Center demo. Transitional: download ALE explainer / trust center |
| 6 | **Failure** | Board packs with unverifiable numbers; tenant bleed; quarantine bypass; float math corruption |
| 7 | **Success** | Defensible ALE narrative, immutable audit trail, tenant-scoped sovereignty, design-partner confidence |

## BrandScript template (fill per campaign)

```
CHARACTER: [Role] at [regulated org type]
PROBLEM: Struggling with [external problem] because [internal struggle]
GUIDE: Ironframe shows [empathy] and proves [authority]
PLAN: 1) Irongate ingest 2) Command Center ALE 3) Ironquery exports
CTA: Schedule a pilot conversation
FAILURE: Regulator/board challenge without dollar-denominated evidence
SUCCESS: Audit-ready quantitative GRC with observable agent workforce
```

## One-liner formula

**Problem + Solution + Result**

> Most GRC platforms show heatmaps that boards cannot defend. Ironframe is the quantitative GRC command post that turns telemetry into dollar-denominated risk and audit-ready exports — so security leaders stop improvising board slides.

Variants by beachhead:

- **Regional BHC:** Subsidiary-isolated command posts with consolidated board ALE.
- **Utility NERC:** Evidence automation aligned to operational technology ingest paths.
- **MSSP enclave:** Per-client tenant boundaries without evidence bleed.

## Marketing Made Simple — five assets

1. **One-liner** — see above; use in email signatures, LinkedIn bio, hero subheads.
2. **Lead generator** — gated ALE calculator brief, NERC/FFIEC pillar PDF, MSSP enclave checklist.
3. **Nurture email series** — 3–5 emails: problem → insight → plan → proof → CTA.
4. **Sales email** — single-thread follow-up after webinar/roundtable with prescribed pilot scope.
5. **Website wireframe** — Header (one-liner) → Stakes → Guide → Plan → CTA → Junk drawer (features) at bottom.

## Website section mapping

| Section | SB7 source | Ironframe content |
|---------|------------|-------------------|
| Header | One-liner | Tagline + demo CTA |
| Stakes | Failure | Cost of heatmap GRC / audit failure |
| Value proposition | Guide + authority | TAS, agents, BigInt integrity |
| Plan | 3 steps | Irongate → Command Center → exports |

| CTA | Direct + transitional | Demo + docs hub |

## Writer agent usage (documentation only)

board-writer applies StoryBrand **structure** to technical docs:

- Open with operator pain (problem), not feature list.
- Position the handbook as guide; reader is hero completing the lab.
- End sections with clear next step (call to action = next lab step).
- Do **not** produce website hero copy or sales emails — see writer agent constraints.

## Trainer agent usage

board-trainer uses SB7 for **clarity**:

- One concept per section; define jargon in glossary blocks.
- Problem → steps → success checklist per lab.
- No pipeline, pricing, or campaign tactics.

## Anti-patterns

- Jargon-heavy hero lines ("synergistic AI-powered GRC transformation").
- Ironframe as hero ("we are the best platform") — customer must be hero.
- Multiple competing CTAs above the fold.
- Claims without Shipped/Pilot/Roadmap labels.
