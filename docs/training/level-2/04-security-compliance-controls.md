# Chapter 4 — Security Architecture & Compliance Controls

> **Track:** LEVEL_2 · **Reading level:** 11th–12th grade plain English · **Release:** `v0.1.0-ga-epic17`  
> **Primary route:** `/audit` · **Lab IDs:** SEC-001

> **Typical role:** CISO / GRC_MANAGER / GLOBAL_ADMIN

## Why this chapter matters

Security controls only help if you can find them and map them to frameworks. This chapter connects `/audit` to the security handbook.

## Learning objectives

When you finish, you can:

- Open `/audit` with an allowed role.
- Find meta-audit panels.
- Open the security and compliance handbook.

## How to get there

1. Sign in with CISO, GRC_MANAGER, or GLOBAL_ADMIN rights.
2. Open `/audit` and scan meta-audit panels.
3. Open `/docs/technical/security-and-compliance`.
4. Map one on-screen control to a handbook note.

## Lab — Control mapping (SEC-001)

1. List three controls or panels you can see on `/audit`.
2. For each, write one plain-English purpose sentence.
3. Cite one handbook path or TAS note that backs the control.
4. If `/audit` redirects unauthorized, record the redirect and stop.

## Check your understanding

- [ ] I know which roles may open `/audit`.
