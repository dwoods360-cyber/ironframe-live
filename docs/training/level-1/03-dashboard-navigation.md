## Why this chapter matters

Fast GRC work depends on knowing where each tool lives. This chapter builds a mental map of the top navigation.

## Learning objectives

When you finish, you can:

- Move between Integrity Hub, Cockpit, Board Report, Evidence, Docs, and Exports.
- Recognize a billing gate when access is blocked.
- Return to `/docs` for handbook help.

## How to get there

1. From a signed-in session, open the top navigation.
2. Visit `/integrity`, `/cockpit`, and `/board-report` in order.
3. If a billing hold appears, read the message. Do not force past it.
4. Open `/docs` and confirm training links load.

## Lab — TopNav tour (NAV-001)

1. Write a table with six rows: Integrity, Cockpit, Board Report, Evidence, Docs, Exports.
2. Click each route once and mark Pass if the page loads.
3. Add the URL path next to each row.
4. Note any PILOT or PREVIEW badge you see. Do not treat those as live deliverables.

## Check your understanding

- [ ] I can reach the six core routes without guessing.
- [ ] I know a billing gate means finish payment or ask a trainer.
- [ ] I know where `/docs` lives.

## Common mistakes

- Skipping the tour and jumping into exports causes room for wrong screenshots.
- PILOT pages show seed data. Use `/exports` for auditor-style files when enabled.

## Glossary

| Term | Plain meaning |
|------|---------------|
| **TopNav** | Primary menu across the top of the app. |
| **Billing gate** | Hold screen when subscription status blocks a feature. |
| **PILOT badge** | Demo / seed surface — not your live production records. |
