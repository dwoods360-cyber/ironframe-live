# NIST SP 800-137 — Information Security Continuous Monitoring (ISCM) — Excerpt

## Chapter 3: The ISCM Process

Organizations shall establish and maintain an ISCM program that provides visibility into assets, vulnerabilities, threats, and security controls.

### 3.1 ISCM Strategy

- Define metrics, monitoring frequency, and escalation paths aligned to organizational risk tolerance.
- Integrate ISCM with incident response and contingency planning.

### 3.2 ISCM Program Implementation

- Continuously monitor information systems and environments of operation.
- Analyze monitoring data to identify adverse events and changes in security posture.
- Respond to findings through prioritized remediation and incident handling.

### 3.3 ISCM Reporting

- Report security status to organizational officials on a defined cadence.
- Support risk management decisions with timely, accurate, and actionable monitoring data.

### 3.4 ISCM Review and Update

- Review and update the ISCM strategy and program at least annually or upon significant change.

## Chapter 4: ISCM Technical Components

- Asset management and inventory.
- Vulnerability management.
- Configuration management.
- Software integrity and malware monitoring.
- Network monitoring and boundary protection.
- Identity and access management monitoring.
- Security event and incident monitoring.
