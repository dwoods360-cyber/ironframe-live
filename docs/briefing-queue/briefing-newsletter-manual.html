1. Pipeline overview

Governance briefings follow a quarantine → review → promote → compile → syndicate model. Drafts never reach the public surface until a human operator signs off.

┌─────────────────────────────────────────────────────────────────────────────┐ │ AUTHORING (quarantined) │ │ docs/briefing-queue/*.md ← manual, IronBoard, or narrate cron │ └───────────────────────────────────┬─────────────────────────────────────────┘ │ human review + Section V citations ▼ ┌─────────────────────────────────────────────────────────────────────────────┐ │ PROMOTION (signed receipt) │ │ scripts/promote-briefing-draft.ts → PostgreSQL published_briefings │ └───────────────────────────────────┬─────────────────────────────────────────┘ │ mirror for web / static RSS / email ▼ ┌─────────────────────────────────────────────────────────────────────────────┐ │ FILESYSTEM LEDGER (public reader compile source) │ │ docs/published-briefings/{slug}.md │ └───────────────┬─────────────────────────────┬───────────────────────────────┘ │ │ ▼ ▼ app/governance-frame/ lib/agents/ironcast/workers/ IronBoard :8082/governance-frame compileNewsletter.ts → HTML │ │ ▼ ▼ public/rss.xml (npm run build:rss) out/governance-frame/newsletters/ │ ▼ GET /api/board/feed?secret=… (live DB RSS — Substack / syndication)
Two persistence layers: Promotion stamps an immutable row in published_briefings (Postgres). The public Next.js reader, static public/rss.xml, and Ironcast newsletter compiler still read docs/published-briefings/. For full surface coverage, complete both Step 3 and Step 4.

3. Step 1 — Create a quarantined draft

1

Choose an authoring path

Manual: Copy docs/briefing-queue/template.md to a dated filename:

docs/briefing-queue/YYYY-MM-DD-draft-{tenant-slug}.md

IronBoard (:8082): Board operator drafts via Command Post; Layer 6 system prompt enforces triad + citations (Ironboard/src/services/boardroomSystemPrompt.ts).

Nightly narrate: Cron writes {date}-draft-{tenant}.md automatically (app/lib/governanceFrame/briefingQueueDraftWriter.ts).

2

Required markdown structure

  • YAML frontmatter — title, tenantId (UUID), activeExposureCents (quoted whole cents), status: "QUARANTINED_DRAFT"
  • ### I. Exposure Vector
  • ### II. Calculated Quantitative Impact — BigInt cent strings only; no float math
  • ### III. Machine-Rule Technical Translation
  • ### V. Sources & Citations — at least one traceable locator before promotion
3

Confirm quarantine (draft must NOT appear publicly)

Reload /governance-frame — queue files must not list. Server logs may show:

[SECURITY AUDIT] Unauthorized compilation attempt blocked for unvetted draft: …

4. Step 2 — Editorial & citation review

1

Fact-check Section V against live telemetry

curl -s "http://127.0.0.1:3000/api/board/shared-context" \
  -H "x-tenant-id: <TENANT_UUID>" | jq .

Cross-reference financials.display.activeTenant, DORA readiness, and TAS baseline (docs/TAS.md).

2

Check exposure escalation flag

If requiresImmediatePromotion: true or exposure ≥ INTERNAL_ALERT_EXPOSURE_THRESHOLD_CENTS (default 5000000 = $50,000.00 USD), treat as urgent HITL review. See .env.example.

3

Reject or redact before promotion

  • Raw CVE-YYYY-NNNN literals (blocked at promotion)
  • Executable HTML / javascript: URIs
  • Missing triad sections I–III or Section V citations

5. Step 3 — Promote to production receipt (PostgreSQL)

Always use the promotion script — never rename/copy queue files directly without validation.
1

Run the signed promotion gate

npx tsx scripts/promote-briefing-draft.ts \
  --file 2026-06-17-draft-medshield.md \
  --slug medshield-sec-update-01 \
  --operator "your.name@corp.example"

On success:

🔒 [MUTATION COMPLETE] Public receipt safely stamped in database.
[AUDIT] {"event":"BRIEFING_PROMOTED",…}
2

Verify live syndication feed (optional)

curl -s "http://127.0.0.1:3000/api/board/feed?secret=$IRONFRAME_CRON_SECRET"

Optional tenant scope: &tenantId=<UUID>

6. Step 4 — Publish to the filesystem ledger

Required for /governance-frame web pages, npm run build:rss, and Ironcast newsletter HTML. Content should match the promoted draft (body without quarantine frontmatter).
1

Write approved markdown to the published ledger

docs/published-briefings/{slug}.md

Use publication frontmatter (example):

---
title: "Medshield Governance Review"
publishedAt: "2026-06-17T12:00:00.000Z"
author: "Executive Intelligence Unit"
classification: "Institutional Governance"
issueNumber: 1
summary: "One-line executive distillation for RSS description."
---
2

Commit & deploy for production

Vercel ships docs/published-briefings/**/* via next.config.ts outputFileTracingIncludes. Production readers update after git deploy — not from local disk alone.

3

Preview locally

npm run dev
# → http://127.0.0.1:3000/governance-frame/{slug}

8. Step 6 — Newsletter HTML (Ironcast / Agent 7)

1

Ensure briefing exists in filesystem ledger

docs/published-briefings/{slug}.md must be present (Step 4).

2

Compile HTML newsletters

From repository root (Node / tsx):

npx tsx -e "
import {
  runGovernanceFramePublicationCompile,
  writeCompiledNewsletterArtifacts,
} from './lib/agents/ironcast/workers/compileNewsletter.ts';

const result = runGovernanceFramePublicationCompile();
const paths = writeCompiledNewsletterArtifacts(result.compiled);
console.log('Compiled', result.compiled.length, 'newsletter(s)');
paths.forEach((p) => console.log(' →', p));
"

Output: out/governance-frame/newsletters/{slug}.html

3

Compile a single slug (programmatic)

Use compileGovernanceFrameNewsletterBySlug('your-slug') from the same module — see tests/unit/governanceFrameEmail.test.ts.

4

Outbound email dispatch

Production transactional email routes through services/ironcast.service.ts and React Email templates under emails/. Governance Frame HTML uses the presentation-only template at lib/agents/ironcast/templates/governanceFrameEmail.ts (no scripts, no inline handlers).

10. Automated nightly path (staggered)

Cadence: Documentation sync at 03:00 → 30-minute settle → GRC narrate at 03:30. This prevents exposure-threshold checks from reading mid-update telemetry.
TimeJobEntry
03:00 local / UTCDoc engine + ironwatch heartbeatscripts/cron_narrate_scheduled.ps1 · Vercel 0 3 * * *
03:30 local / UTCGRC triad narrate + queue draftscripts/cron_narrate_api_scheduled.ps1 · POST /api/cron/narrate
1

Register Windows tasks (one-time)

powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\register-nightly-cron-tasks.ps1
2

Trigger narrate manually (smoke)

curl -X POST "http://127.0.0.1:3000/api/cron/narrate" \
  -H "Authorization: Bearer $IRONFRAME_CRON_SECRET" \
  -H "x-tenant-id: <TENANT_UUID>"

Scheduled local run: scripts/cron_narrate_api_scheduled.ps1 (03:30). Doc engine log: scripts/cron_narrate.log · API log: logs/cron_narrate_log.txt.

3

What narrate produces (03:30 window)

  • governance_frame_triad_snapshots row — app/lib/reports/narrateGovernanceTriad.ts
  • cron_job_artifact audit row
  • Quarantined draft in docs/briefing-queue/{date}-draft-{tenant}.md

Narrate does not auto-publish. A human must still run Step 3 (and Step 4 for web/RSS/email).

12. Troubleshooting

Promotion blocked

  • Read [ERROR] lines from promote-briefing-draft.ts output.
  • Add Section V citations and triad sections; remove CVE literals.
  • Ensure frontmatter includes valid tenantId UUID.

Briefing missing on /governance-frame

  • Confirm file exists at docs/published-briefings/{slug}.md (Step 4).
  • DB promotion alone does not update the filesystem reader.
  • On Vercel, commit and redeploy so traced files ship with the build.

Empty RSS

  • Static: run npm run build:rss after adding published markdown.
  • Live API: confirm published_briefings rows exist and feed secret matches IRONFRAME_CRON_SECRET.

Newsletter HTML empty or missing slug

  • Slug must exist in docs/published-briefings/.
  • Run compile with explicit slugs: ['your-slug'] option to runGovernanceFramePublicationCompile.
